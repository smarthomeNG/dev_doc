    Die aktuellste Version (0.4.19, released März 2019) des benötigten Packages 'openzwave' ist nicht kompatibel mit Python 3.9



Anforderungen
=============




 - Minimum SmartHomeNG Version: **1.3**
 - Maximum Python Version: **3.8**

Konfiguration
=============


Im folgenden ist beschrieben, wie das Plugin **zwave** konfiguriert wird. Außerdem ist im folgenden beschrieben, wie das Plugin in den Item Definitionen genutzt werden kann.


=========
Parameter
=========


Das Plugin verfügt über folgende Parameter, die in der Datei ``../etc/plugin.yaml`` konfiguriert werden:


config_path
===========


Der Ordner in dem die Konfiuration liegt

 - Datentyp: **str**
 - Standardwert: **/etc/openzwave/**


device
======


Gerätedatei des Z-Wave Garätes

 - Datentyp: **str**
 - Standardwert: **/dev/ttyUSB0**


logfile
=======


set the name of the logfile

 - Datentyp: **str**
 - Standardwert: **OZW.log**


loglevel
========


set the loglevel

 - Datentyp: **str**
 - Standardwert: **Info**


sec_strategy
============


Definiert die Sicherheitsstrategie : SUPPORTED|ESSENTIAL|CUSTOM

 - Datentyp: **str**
 - Standardwert: **SUPPORTED**


zlogging
========


True to activate logs, False to disable

 - Datentyp: **bool**
 - Standardwert: **False**


==============
Item Attribute
==============


Das Plugin unterstützt folgende Item Attribute, die in den Dateien im Verzeichnis  ``../items`` verwendet werden:


zwave_node
==========


connects the item to the defined node

 - Datentyp: **str**


zwave_value
===========


connects the item to the defined value

 - Datentyp: **str**

Beschreibung
============

Plugin zur Ansteuerung von SmartHomeNG via Amazon Echo bzw. Alexa


Anforderungen
=============




 - Minimum SmartHomeNG Version: **1.5.2**

Konfiguration
=============


Im folgenden ist beschrieben, wie das Plugin **alexa4p3** konfiguriert wird. Außerdem ist im folgenden beschrieben, wie das Plugin in den Item Definitionen genutzt werden kann.


=========
Parameter
=========


Das Plugin verfügt über folgende Parameter, die in der Datei ``../etc/plugin.yaml`` konfiguriert werden:


service_port
============


Portnummer auf dem das Plugin die Anfragen von Amazon-Alexa erwartet

 - Datentyp: **int**
 - Standardwert: **9000**


==============
Item Attribute
==============


Das Plugin unterstützt folgende Item Attribute, die in den Dateien im Verzeichnis  ``../items`` verwendet werden:


alexa_actions
=============


Alexa Aktionen die für diese Gerät bereitgestellt werden

 - Datentyp: **str**


alexa_auth_cred
===============


Benutzer und Passwort für Kamerastreams als String im Format USER:PWD

 - Datentyp: **str**


alexa_camera_imageUri
=====================


URL für das Standbild einer Kamera

 - Datentyp: **str**


alexa_color_value_type
======================


Farbsystem für RGB/HSV Leuchten, Wert = RGB oder HSV, Standardwert ist RGB

 - Datentyp: **str**


alexa_csc_uri
=============


URLs der Kamerastreams als String im Format {"Stream1":"192.168.178.9","Stream2":"192.168.178.9/2"},"Stream3":"...."

 - Datentyp: **str**


alexa_description
=================


Alexa Gerätename um das Gerät anzusprechen als Standard-Wert wird alexa_name verwendet

 - Datentyp: **str**


alexa_device
============


interner Gerätename um das Gerät in smarthomeNG zu indentifizieren (endpoint)

 - Datentyp: **str**


alexa_icon
==========


Alexa Kategorie/Symbol welche in der Alexa-App angezeigt wird, Standard-Wert SWITCH

 - Datentyp: **str**


alexa_item_range
================


Wertbereich in dem das Gerät eingestellt werden kann z.B. 0-255 oder 0-100, default ist 0-255

 - Datentyp: **str**


alexa_item_turn_on
==================


Wert der beim Einschalten zum Item gesendet werden soll

 - Datentyp: **str**


alexa_name
==========


Alexa Gerätename um das Gerät anzusprechen as default fall back to alexa_device

 - Datentyp: **str**


alexa_proactivelyReported
=========================


legt fest ob die Statuswerte bei Änderung an Amazon / Alexa übermittelt werden sollen, default FALSE

 - Datentyp: **bool**


alexa_retrievable
=================


legt fest ob die Statuswerte durch Amazon/Alexa abgefragt werden können, default FALSE

 - Datentyp: **bool**


alexa_stream_(1)1(2)(3)
=======================


Streamdefinition (Codecs, Aufloesung, ...)für Kamerastreams, es sind drei Streams moeglich (alexa_stream_1, alexa_stream_2, alexa_stream_3)

 - Datentyp: **str**


alexa_thermo_config
===================


moegliche Modi für einen Raumthermostaten, möglich Werte -  0:AUTO 1:HEAT 2:OFF 3:ECO 4:ECO, Standardwert ist = 0:AUTO

 - Datentyp: **str**


alexa_types
===========


nur für Payload-V2 benötigt

 - Datentyp: **str**

Anforderungen

Konfiguration
=============


Im folgenden ist beschrieben, wie das Plugin **drexelundweiss** konfiguriert wird. Außerdem ist im folgenden beschrieben, wie das Plugin in den Item Definitionen genutzt werden kann.


=========
Parameter
=========


Das Plugin verfügt über folgende Parameter, die in der Datei ``../etc/plugin.yaml`` konfiguriert werden:


LU_ID
=====


ID für Lüftung.

 - Datentyp: **int**
 - Standardwert: **130**


PANEL_ID
========


ID für Touchpanel.

 - Datentyp: **int**
 - Standardwert: **120**


WP_ID
=====


ID für Heizung.

 - Datentyp: **int**
 - Standardwert: **140**


busmonitor
==========


Busmonitor.

 - Datentyp: **bool**
 - Standardwert: **False**


device
======


ID des Geräts, falls die Info nicht automatisch ausgelesen werden kann.

 - Datentyp: **int**
 - Standardwert: **0**


retrylimit
==========


Maximale Versuche, einen Wert einzulesen.

 - Datentyp: **int**
 - Standardwert: **100**


tty
===


Beim Nutzen einer RS232/USB Schnittstelle ist die serielle Schnittstelle anzugeben.

 - Datentyp: **str**


==============
Item Attribute
==============


Das Plugin unterstützt folgende Item Attribute, die in den Dateien im Verzeichnis  ``../items`` verwendet werden:


DuW_LU_register
===============


Register für Lüftung entsprechend Textdatei

 - Datentyp: **int**


DuW_PANEL_register
==================


Register für Touch Panel entsprechend Textdatei

 - Datentyp: **int**


DuW_WP_register
===============


Register für Wärmepumpe entsprechend Textdatei

 - Datentyp: **int**

Beschreibung
============

Unterstützung für Round Robin Datenbanken (rrdtool)


Anforderungen
=============




 - Minimum SmartHomeNG Version: **1.5**

Konfiguration
=============


Im folgenden ist beschrieben, wie das Plugin **rrd** konfiguriert wird. Außerdem ist im folgenden beschrieben, wie das Plugin in den Item Definitionen genutzt werden kann.


=========
Parameter
=========


Das Plugin verfügt über folgende Parameter, die in der Datei ``../etc/plugin.yaml`` konfiguriert werden:


rrd_dir
=======


Verzeichnis der rrd Datenbanken. Wenn leer, wird der SmartHomeNG Basis Pfad + /var/rrd genutzt

 - Datentyp: **str**


step
====


Zeit zwischen zwei Datenpunkten. Default ist 300 Sekunden.

 - Datentyp: **int**
 - Standardwert: **300**


==============
Item Attribute
==============


Das Plugin unterstützt folgende Item Attribute, die in den Dateien im Verzeichnis  ``../items`` verwendet werden:


rrd
===


Wenn auf 'yes' gesetzt, werden die Werte des Items in die rrd Datenbank geschrieben.  Wenn auf 'init' gesetzt, wird zusätzlich beim Start von SmartHomeNG der Wert aus rrd gelesen.   Wenn für dieses Item ebenfalls ein Attribut 'sqlite' oder 'database' gesetzt ist,  so ist die Wahrscheinlichkeit groß, das eines der Plugins nicht funktioniert. Um dieses Problem zum umgehen kann das Attribut 'rrd_no_series' gesetzt werden.

 - Datentyp: **str**

 - Mögliche Werte:

   - 
   - **yes**
   - **init**


rrd_ds_name
===========


Alternativen Datenquellennamen festlegen. Wird dieser Name festgelegt, so wird nicht der Item Name für den Datenquellennamen verwendet

 - Datentyp: **str**


rrd_max
=======


Wird dieses Attribut auf True gesetzt, wird auch der Maximumwert in die Datenbank geschrieben. Standard ist False

 - Datentyp: **bool**
 - Standardwert: **False**


rrd_min
=======


Wird dieses Attribut auf True gesetzt, wird auch der Minimumwert in die Datenbank geschrieben. Standard ist False

 - Datentyp: **bool**
 - Standardwert: **False**


rrd_no_series
=============


Wird dieses Attribut auf True gesetzt, werden Daten in die Datenbank geschrieben, können aber nicht durch eine Visu abgerufen werden.

 - Datentyp: **bool**
 - Standardwert: **False**


rrd_step
========


Zeit zwischen zwei Datenpunkten. Default ist der Wert vom Plugin Parameter rrd_step

 - Datentyp: **int**


rrd_type
========


Gibt die Art der Datenreihe in der Datenbank an: gauge für Temperaturen, counter für Zähler

 - Datentyp: **str**

 - Mögliche Werte:

   - 
   - **gauge**
   - **counter**
